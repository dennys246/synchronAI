# fNIRS Signal Types and Training

This project supports training diffusion models on two different types of fNIRS signals:

## 1. Hemodynamic Signals (Default)

**What it is:** Raw hemoglobin concentration changes (HbO and HbR) measured by fNIRS sensors.

**Files:**
- `.snirf` files (raw data)
- `.fif` files WITHOUT "Deconvolved" in the name

**Training script:** `scripts/generative_pretrain.sh`

**Output directory:** `runs/fnirs_diffusion/`

**Command:**
```bash
bash scripts/generative_pretrain.sh
# Or with options:
bash scripts/generative_pretrain.sh --epochs 100 --max-recordings 200
```

**Direct Python call:**
```bash
python -m synchronai.main --fnirs --train diffusion \
  --signal-type hemodynamic \
  --data-dir /path/to/data \
  --save-dir runs/fnirs_diffusion
```

---

## 2. Neural Activity Signals

**What it is:** Deconvolved neural activity estimates generated by HRfunc preprocessing. These represent the underlying neural activity that causes the hemodynamic response.

**Files:**
- `.fif` files WITH "Deconvolved" or "_Deconvolved" in the name
- These are processed outputs from HRfunc deconvolution

**Training script:** `scripts/neural_pretrain.sh`

**Output directory:** `runs/fnirs_neural_diffusion/`

**Command:**
```bash
bash scripts/neural_pretrain.sh
# Or with options:
bash scripts/neural_pretrain.sh --epochs 100 --max-recordings 200
```

**Direct Python call:**
```bash
python -m synchronai.main --fnirs --train diffusion \
  --signal-type neural \
  --data-dir /path/to/data \
  --save-dir runs/fnirs_neural_diffusion
```

## File Discovery Behavior

The lazy discovery system automatically filters files based on `--signal-type`:

### Hemodynamic Mode (`--signal-type hemodynamic`, default)
- ✅ Discovers: `1114_Flanker.snirf`, `subject_task.fif`
- ❌ Skips: `1114_Flanker_Deconvolved.fif`, `subject_task_deconvolved.fif`

### Neural Mode (`--signal-type neural`)
- ✅ Discovers: `1114_Flanker_Deconvolved.fif`, `subject_task_deconvolved.fif`
- ❌ Skips: `1114_Flanker.snirf`, `subject_task.fif` (raw files)

This ensures:
1. No file conflicts or duplicate training
2. Clear separation between hemodynamic and neural models
3. Appropriate data for each model type

## Common Options (Both Scripts)

| Option | Default | Description |
|--------|---------|-------------|
| `--epochs N` | 0 (unlimited) | Number of training epochs (0 = train forever until Ctrl+C) |
| `--max-recordings N` | 0 (unlimited) | Maximum number of recordings to train on |
| `--recordings-per-batch N` | 4 | Number of recordings to load into memory at once |
| `--lr FLOAT` | 1e-4 | Learning rate |
| `--verbosee` | off | Enable verbose logging |
| `--disable-numba` | off | Disable Numba JIT compilation (for debugging) |

## Implementation Details

### File Filtering Logic

Located in `src/synchronai/training/diffusion/train.py`:

```python
# In LazyFnirsDiscovery._iter_fnirs_paths()
is_deconvolved = "_Deconvolved" in filename or "deconvolved" in filename.lower()

if self.signal_type == "hemodynamic" and is_deconvolved:
    # Skip neural activity files when training hemodynamic model
    continue
elif self.signal_type == "neural" and not is_deconvolved:
    # Skip raw hemodynamic files when training neural model
    continue
```

### File Existence Checks

All discovered files are verified to exist before yielding:

```python
if file_path.exists() and file_path.is_file():
    yield str(file_path)
```

This prevents issues with:
- Broken symlinks
- Missing files
- Files deleted between discovery and loading

## Troubleshooting

**Issue:** Training fails with "FileNotFoundError" for `_Deconvolved.fif` files

**Solution:** You're likely running `generative_pretrain.sh` which trains on hemodynamic signals. Deconvolved files are now automatically skipped. This error should not occur with the updated code.

**Issue:** No files discovered in neural mode

**Cause:** No deconvolved files exist in your dataset directories

**Solution:** Either:
1. Generate deconvolved files using HRfunc with `--deconvolution` flag
2. Train on hemodynamic signals instead using `generative_pretrain.sh`

**Issue:** Training on wrong signal type

**Solution:** Check which script you're using:
- `generative_pretrain.sh` → hemodynamic (raw HbO/HbR)
- `neural_pretrain.sh` → neural (deconvolved activity)
